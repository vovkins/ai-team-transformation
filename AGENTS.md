# Инструкции для AI-агентов

Этот репозиторий хранит русскоязычное руководство по внедрению AI-агентов в
команду разработки.

Пишите просто. Текст должен быть понятен человеку без знания внутренних терминов
`Zabr`.

## Как читать репозиторий

1. `README.md` — главный документ и источник истины.
2. `raw-notes.md` — черновые заметки. Это не правила процесса.
3. Новые документы добавляйте только если раздел становится слишком большим для
   `README.md`.

## Правила редактирования

- Основной текст должен быть на простом русском языке.
- Сложные термины можно оставлять только тогда, когда без них нельзя, и рядом
  нужно дать понятное объяснение.
- Не создавайте несколько разных списков навигации. Один маршрут чтения должен
  жить в одном месте.
- Разделяйте общие правила команды, правила конкретного проекта и правила
  конкретной задачи.
- Если добавляете роль, навык или шаблон, укажите:
  - когда это использовать;
  - кто владелец;
  - что нужно на входе;
  - что должно получиться на выходе;
  - как проверить результат;
  - где требуется решение человека.
- Не делайте утверждений о внешних каталогах, инструментах или сервисах без
  проверки.
- Идеи можно временно складывать в `raw-notes.md`, но принятые правила должны
  попадать в `README.md` или отдельный связанный документ.

## Проверка

Перед завершением правки запустите:

```bash
git diff --check
```

Если менялись таблицы, списки или блоки кода, дополнительно проверьте глазами,
что Markdown читается нормально.
